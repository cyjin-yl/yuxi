## 读题

这道题的核心是"配对"——从一堆数字中选出若干对，每对要满足特定条件，然后让总收益最大。

具体条件有三层：

**第一层：比值是质数。** 两个数字 &#36;a_i, a_j&#36;，把较大的除以较小的，结果必须是一个质数。比如 &#36;6&#36; 和 &#36;2&#36;，&#36;6/2 = 3&#36;，&#36;3&#36; 是质数，满足条件。&#36;12&#36; 和 &#36;4&#36;，&#36;12/4 = 3&#36;，也满足。但 &#36;12&#36; 和 &#36;6&#36;，&#36;12/6 = 2&#36;，也满足——所以这个条件比看起来要宽松一些。

**第二层：收益计算。** 配对 &#36;i,j&#36; 的收益是 &#36;b_i \times c_j + b_j \times c_i&#36;。注意这里 &#36;b&#36; 是数量，&#36;c&#36; 是单价。这个公式的本质是：&#36;i&#36; 的 &#36;b_i&#36; 个单位都卖给 &#36;j&#36; 的单价 &#36;c_j&#36;，同时 &#36;j&#36; 的 &#36;b_j&#36; 个单位都卖给 &#36;i&#36; 的单价 &#36;c_i&#36;。

**第三层：每个数字只能用一次。** 题目说"每两个城市只能进行一次交易"，意思是每对 &#36;(i, j)&#36; 只能配对一次，但一个城市 &#36;i&#36; 可以和多个不同城市配对——只要它还有剩余粮食 &#36;b_i&#36;。

读题到这里，我的第一直觉是：**这是一个匹配问题**。但和普通二分图匹配不同，收益不是简单的边权相加，而是和 &#36;b, c&#36; 两个属性都相关。而且"比值是质数"这个条件需要数论处理。

## 思考过程

从读题到确定方法，我的推理链路是这样的：

![](/graphs/p4068-thinking.svg)

**第一步：条件等价变形。**

"&#36;a_i / a_j&#36; 或 &#36;a_j / a_i&#36; 是质数"这个条件，等价于什么？

设 &#36;g = \gcd(a_i, a_j)&#36;，令 &#36;x_i = a_i / g&#36;，&#36;x_j = a_j / g&#36;。那么 &#36;x_i&#36; 和 &#36;x_j&#36; 互质。

条件"比值是质数"就是：&#36;x_i / x_j&#36; 或 &#36;x_j / x_i&#36; 是质数。由于两者互质，这意味着 &#36;x_i&#36; 和 &#36;x_j&#36; 中**恰好一个是 &#36;1&#36;，另一个是质数**。

举例：&#36;a_i = 6, a_j = 3&#36;，&#36;g = \gcd(6, 3) = 3&#36;，&#36;x_i = 2, x_j = 1&#36;。&#36;2&#36; 是质数，&#36;1&#36; 不是，满足条件。

再举：&#36;a_i = 12, a_j = 6&#36;，&#36;g = 6&#36;，&#36;x_i = 2, x_j = 1&#36;。同样满足。

再举：&#36;a_i = 12, a_j = 4&#36;，&#36;g = 4&#36;，&#36;x_i = 3, x_j = 1&#36;。&#36;3&#36; 是质数，满足。

再举：&#36;a_i = 12, a_j = 8&#36;，&#36;g = 4&#36;，&#36;x_i = 3, x_j = 2&#36;。两个都不是 &#36;1&#36;，不满足。

**第二步：质因数分解与二分图性质。**

这是本题最关键的观察：**一个数字 &#36;a&#36;，设它的质因数分解为 &#36;a = p_1^&#123;e_1&#125; p_2^&#123;e_2&#125; \cdots p_k^&#123;e_k&#125;&#36;，令总质因数个数（计重数）&#36;cnt = e_1 + e_2 + \cdots + e_k&#36;。**

如果两个数字 &#36;a_i, a_j&#36; 满足"比值是质数"的条件，那么它们的 &#36;cnt&#36; 值**奇偶性一定不同**。

为什么？设 &#36;g = \gcd(a_i, a_j)&#36;，&#36;x_i = a_i/g&#36;，&#36;x_j = a_j/g&#36;。其中一个为 &#36;1&#36;，另一个为质数 &#36;p&#36;。

- 若 &#36;x_i = 1, x_j = p&#36;：&#36;a_i = g&#36;，&#36;a_j = g \times p&#36;。&#36;a_i&#36; 的质因数个数是 &#36;cnt(g)&#36;，&#36;a_j&#36; 的是 &#36;cnt(g) + 1&#36;。奇偶性不同。
- 若 &#36;x_i = p, x_j = 1&#36;：对称，同样奇偶性不同。

所以，**按 &#36;cnt&#36; 的奇偶性把数字分成两组，只有不同组的数字才可能配对**。这天然构成一个二分图。

**第三步：建模为费用流。**

二分图的边权是 &#36;b_i \times c_j + b_j \times c_i&#36;。我们要求最大权匹配，但这里的"匹配"不是简单的每条边选或不选——每个节点有容量限制 &#36;b_i&#36;（粮食量）。

正确的建图方式：

1. 源点 &#36;S&#36; 连向所有左组节点 &#36;i&#36;，容量 &#36;b_i&#36;，费用 &#36;0&#36;
2. 右组节点 &#36;j&#36; 连向汇点 &#36;T&#36;，容量 &#36;b_j&#36;，费用 &#36;0&#36;
3. 二分图中所有合法边 &#36;(i, j)&#36;，容量 &#36;\infty&#36;（或一个足够大的数），费用 &#36;b_i \times c_j + b_j \times c_i&#36;
4. 求 **最大费用最大流**——用 SPFA 找最长增广路，不断增广直到没有正权路径

为什么容量设为 &#36;\infty&#36;？因为一条边代表的是"这个配对可以产生这么多收益"，实际能增广多少取决于源点和汇点的容量限制。

## 代码实现

```cpp
#include <bits/stdc++.h>
using namespace std;

typedef long long ll;

const int MAXN = 305;
const int MAXE = 100000;
const ll INF = 1e18;

// --- 费用流板子 ---

struct Edge {
    int to, nxt, cap;
    ll cost;
} e[MAXE];

int head[MAXN], ecnt = 1, S, T;
int cnt[MAXN], n;
ll a[MAXN], b[MAXN], c[MAXN];
ll dist[MAXN], pre[MAXN];
bool inq[MAXN];
queue<int> q;

void addEdge(int u, int v, int cap, ll cost) {
    e[++ecnt] = {v, head[u], cap, cost};
    head[u] = ecnt;
    e[++ecnt] = {u, head[v], 0, -cost};
    head[v] = ecnt;
}

// SPFA 求最长路（最大费用流）
bool spfa() {
    for (int i = 0; i <= T; i++) {
        dist[i] = -INF;
        pre[i] = 0;
        inq[i] = false;
    }
    dist[S] = 0;
    q.push(S);
    inq[S] = true;
    while (!q.empty()) {
        int u = q.front();
        q.pop();
        inq[u] = false;
        for (int i = head[u]; i; i = e[i].nxt) {
            int v = e[i].to;
            if (e[i].cap > 0 && dist[v] < dist[u] + e[i].cost) {
                dist[v] = dist[u] + e[i].cost;
                pre[v] = i;
                if (!inq[v]) {
                    inq[v] = true;
                    q.push(v);
                }
            }
        }
    }
    return dist[T] > 0; // 只要还有正权路径就继续增广
}

int main() {
    scanf("%d", &n);
    S = 0, T = n + 1;

    // --- 读入 + 质因数分解 ---
    for (int i = 1; i <= n; i++) {
        scanf("%lld%lld%lld", &a[i], &b[i], &c[i]);
        // 计算质因数个数（计重数）
        ll temp = a[i];
        for (int d = 2; d * d <= temp; d++) {
            while (temp % d == 0) cnt[i]++, temp /= d;
        }
        if (temp > 1) cnt[i]++;
    }

    // --- 源点/汇点连边 ---
    for (int i = 1; i <= n; i++) {
        if (cnt[i] & 1)  // 奇数：左组，源点连过来
            addEdge(S, i, (int)b[i], 0);
        else             // 偶数：右组，连向汇点
            addEdge(i, T, (int)b[i], 0);
    }

    // --- 二分图边 ---
    for (int i = 1; i <= n; i++)
        for (int j = i + 1; j <= n; j++) {
            ll g = std::gcd(a[i], a[j]);
            ll xi = a[i] / g, xj = a[j] / g;
            // 检查：一个为1，另一个为质数
            bool ok = (xi == 1 && xj > 1) || (xj == 1 && xi > 1);
            if (!ok) continue;
            // 进一步确认非1的那个是质数
            ll p = xi == 1 ? xj : xi;
            bool isPrime = true;
            for (int d = 2; d * d <= p; d++) {
                if (p % d == 0) { isPrime = false; break; }
            }
            if (!isPrime) continue;

            // 奇偶性不同才加边
            if ((cnt[i] & 1) == (cnt[j] & 1)) continue;

            ll val = b[i] * c[j] + b[j] * c[i];
            if (cnt[i] & 1)
                addEdge(i, j, 1e9, val);  // 左→右
            else
                addEdge(j, i, 1e9, val);  // 左→右（保证方向）
        }

    // --- 增广直到无正权路径 ---
    ll ans = 0;
    while (spfa()) {
        ll flow = 1e9;
        for (int i = pre[T]; i; i = pre[e[i ^ 1].to])
            flow = min(flow, (ll)e[i].cap);
        ans += flow * dist[T];
        for (int i = pre[T]; i; i = pre[e[i ^ 1].to])
            e[i].cap -= flow, e[i ^ 1].cap += flow;
    }

    printf("%lld\n", ans);
    return 0;
}
```

## 调试记录

实际写代码时踩了几个坑，值得记录：

### 坑一：质数判断遗漏

第一版代码只检查了"一个为 &#36;1&#36;，另一个大于 &#36;1&#36;"，但没有确认那个大于 &#36;1&#36; 的数真的是质数。比如 &#36;a_i = 12, a_j = 2&#36;，&#36;g = 2&#36;，&#36;x_i = 6, x_j = 1&#36;。&#36;6&#36; 不是质数，但第一版会误判为合法边。

**修复：** 加上质数判断。

### 坑二：奇偶性检查

第二版发现一个问题：即使 &#36;x_i = 1, x_j = p&#36;（&#36;p&#36; 是质数），如果 &#36;cnt[i]&#36; 和 &#36;cnt[j]&#36; 奇偶性相同，也不应该加边。虽然理论上这种情况不会出现（前面已经证明了），但作为防御性编程，加上检查更安全。

**修复：** 加 `if ((cnt[i] & 1) == (cnt[j] & 1)) continue;`。

### 坑三：增广方向

费用流的增广是找"最长路"（正权最大），而不是最短路径。SPFA 里要用 `dist[v] < dist[u] + cost` 而不是 `dist[v] > dist[u] + cost`。一开始搞反了导致 WA。

**修复：** 确认比较方向。

### 坑四：`__gcd` 的兼容性

`std::gcd` 是 C++17 才有的。如果编译器不支持，可以用 `__gcd`（GCC 扩展）或者手写。

## 复杂度分析

- 质因数分解：&#36;O(n \cdot \sqrt&#123;a_i&#125;)&#36;
- 建图：&#36;O(n^2)&#36;
- 费用流：SPFA 每次 &#36;O(E)&#36;，增广次数取决于具体数据，本题 &#36;n \le 300&#36;，完全可过

总时间复杂度 &#36;O(n^2 + \text&#123;SPFA&#125; \times E)&#36;，空间 &#36;O(n + E)&#36;。

## 总结

这道题的关键在于：

1. **将数论条件转化为图论结构**——质因数个数的奇偶性直接给出二分图划分
2. **费用流建模**——把"每个节点有容量限制"的匹配问题转化为最大费用最大流
3. **SPFA 最长路增广**——只要还有正权增广路就继续，保证全局最优

这道题的巧妙之处在于，"比值是质数"这个看似复杂的条件，通过质因数分解的视角，变得非常直观。
